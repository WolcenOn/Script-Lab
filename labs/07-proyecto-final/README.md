# Lab 07 - Proyecto final

## Objetivo

Crear un informe completo de mantenimiento que una los laboratorios anteriores.

---

## Relación con la guía

Consulta antes:

- Proyecto final de mantenimiento.
- Monitorización.
- Runbooks e incidencias.
- Integración con Script-Lab.

---

## Escenario

Debes revisar un sistema de laboratorio y entregar un informe técnico con diagnóstico y recomendaciones.

---

## Apartados obligatorios

El informe debe incluir:

1. Fecha y equipo.
2. Sistema operativo.
3. Recursos: CPU, memoria y disco.
4. Red y conectividad.
5. Servicios relevantes.
6. Procesos con mayor consumo.
7. Logs o eventos recientes.
8. Usuarios o grupos relevantes.
9. Estado de copias o propuesta de backup.
10. Conclusiones y recomendaciones.

---

## Criterios de evaluación

| Criterio | Nivel esperado |
|---|---|
| Orden | Informe estructurado y claro |
| Diagnóstico | Explica qué se revisa y por qué |
| Evidencias | Incluye salidas relevantes |
| Conclusiones | Resume estado del sistema |
| Recomendaciones | Propone mejoras razonables |
| Automatización | Incluye propuesta de script o mejora futura |

---

## Preguntas finales

1. ¿Cuál es el estado general del sistema?
2. ¿Qué revisarías primero si hubiera una incidencia?
3. ¿Qué parte automatizarías?
4. ¿Qué riesgo has detectado?
5. ¿Qué mejorarías en una segunda versión?
